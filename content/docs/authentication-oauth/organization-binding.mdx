---
title: Organization Binding
description: Learn about organization binding in Zoho Desk OAuth.
---

Any OAuth token generated is linked to a Zoho Desk organization. This step of associating the token and the organization — which happens in the backend while getting authorization consent from the end user — is called organization binding.

If the end-user has only one Zoho Desk organization (portal), the OAuth token is linked to it automatically. If they have multiple organizations, they must manually select the one with which the token must be linked, as illustrated in the following image.
![Generated Token](/images/APIDocument_OAuthOrgConSentDemo.b8c4a5260162aa9b9678.png)

After they click the **Submit** button, the OAuth token is generated with the organization information bound to it.

<Callout>
Zoho Desk APIs automatically extract organization information from OAuth tokens. Therefore, it is not necessary to add the extra `orgId` query parameter in an API call or pass it in the API's header. However, if you choose to do so, make sure that the value of `orgId` is the same as that of the portal bound with the OAuth token. If the IDs do not match, the API call fails and the [`OAUTH_ORG_MISMATCH`](/docs/getting-started/errors#oauth_org_mismatch) error is returned.
</Callout>
