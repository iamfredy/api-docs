title: Get colliding users action and chat status in a ticket
description: >-
  This API should be called after ticket is rendered in detailed view. This
  returns collision users (agent Id, their action in ticket and author details)
  and chat status of that collision. 'chatId' returns the uniqueId of the chat
  that belongs to a collision session. This chatId should be used only for
  publishing messages within the chat members. 'chatUsers' returns the agentIds
  of current users in chat. The colliding users action is identified by 2
  arrays, 'collidingUsers' returns the agentIds of users colliding in ticket and
  'usersAction' returns the action of corresponding colliding users in the same
  order respectively. The values 'collidingUsers' and 'usersAction' will be
  deprecated soon (DONOT USE THESE VALUES)
